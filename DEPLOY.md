# DSA Tracker — Deployment Guide

Complete guide to go from local → GitHub → Render → Live website with automatic daily sync.

---

## Architecture Overview

```
User opens website
      │
      ▼
Clicks "Connect LeetCode"
      │
      ▼
Chrome Extension popup opens
      │
Reads browser cookies automatically
  ├── LEETCODE_SESSION
  └── csrftoken
      │
      ▼
POST /save_cookie  (Flask on Render)
      │
      ▼
PostgreSQL DB (Render managed)
      │
      ▼
Daily Render Cron → scheduler.py
      │
Uses stored cookie to fetch submissions
      │
      ▼
Google Sheet updated ✅
```

---

## Step 1 — Prepare the repo for GitHub

```bash
# In your project folder
git init
git add .
git commit -m "initial: automation12 with extension + PostgreSQL"

# Create a new GitHub repo at github.com/new (name: dsa-tracker)
git remote add origin https://github.com/YOUR_USERNAME/dsa-tracker.git
git push -u origin main
```

> **Security:** `dsa_tracker.db`, `*.json` credential files, and `.env` are listed in `.gitignore` — they never go to GitHub.

---

## Step 2 — Create a Render account

1. Go to [render.com](https://render.com) and sign up (free).
2. Connect your GitHub account.

---

## Step 3 — Deploy via Blueprint (render.yaml)

1. In Render dashboard → **New** → **Blueprint**.
2. Choose your `dsa-tracker` GitHub repo.
3. Render reads `render.yaml` and creates:
   - **Web Service** — `dsa-tracker` (Flask app)
   - **Cron Job** — `dsa-tracker-daily-sync` (runs `scheduler.py` at 02:00 UTC)
   - **PostgreSQL DB** — `dsa-tracker-db` (free 1 GB)

---

## Step 4 — Set Environment Variables in Render

Go to **dsa-tracker** web service → **Environment** tab and add:

| Variable | Value |
|---|---|
| `GOOGLE_SERVICE_JSON` | Paste the full content of your service account `.json` file |
| `SMTP_USER` | Your Gmail address |
| `SMTP_PASS` | Your Gmail [App Password](https://myaccount.google.com/apppasswords) |
| `FROM_EMAIL` | Same as SMTP_USER |
| `APP_URL` | `https://dsa-tracker.onrender.com` (your Render URL) |

> `SECRET_KEY` and `DATABASE_URL` are auto-generated/injected by `render.yaml` — you don't need to set them manually.

Do the same for the **cron job** service (just `DATABASE_URL` and `GOOGLE_SERVICE_JSON` are needed there).

---

## Step 5 — Install the Chrome Extension

The extension folder is `extension/` in the project root.

1. Open Chrome → `chrome://extensions/`
2. Enable **Developer mode** (top right toggle).
3. Click **Load unpacked**.
4. Select the `extension/` folder from the project.
5. Pin the extension to your toolbar.

> **Note for other users:** You'll need to distribute the extension manually (or publish it to Chrome Web Store). Each user loads it in the same way.

---

## Step 6 — First-Time User Flow

1. User registers on the website.
2. Admin approves / verifies the account.
3. User logs in to **leetcode.com** in the same browser.
4. User clicks the DSA Tracker extension icon.
5. In the popup:
   - Enter the Render URL: `https://dsa-tracker.onrender.com`
   - Click **Connect LeetCode**.
6. Extension reads `LEETCODE_SESSION` + `csrftoken` cookies and POSTs them to `/save_cookie`.
7. Flask saves them to PostgreSQL.
8. ✅ Done — daily sync runs automatically.

---

## Step 7 — What happens daily (automatic)

- Render Cron fires `scheduler.py` at 02:00 UTC every day.
- For each active user: fetches latest CF/LC/AC submissions and updates DB + Google Sheet.
- If a cookie has expired (LeetCode returns 401/403): `cookie_expiry=1` is set in DB.
- The website shows a **"Reconnect LeetCode"** banner to that user.
- User just clicks Connect in the extension again — one click, done.

---

## Cookie Expiry Flow

```
Scheduler runs
      │
LeetCode returns 401/403
      │
      ▼
cookie_expiry = 1  in DB
      │
      ▼
User sees "Reconnect LeetCode" banner on dashboard
      │
User clicks extension → Connect LeetCode
      │
      ▼
New cookie saved → cookie_expiry = 0
      │
      ▼
Next scheduler run syncs normally ✅
```

## Environment Variables Reference

| Variable | Required | Description |
|---|---|---|
| `DATABASE_URL` | Yes | Required everywhere — must point to a PostgreSQL database. |
| `SECRET_KEY` | Yes | Auto-generated by Render. Set manually for local dev. |
| `GOOGLE_SERVICE_JSON` | Yes | Full JSON string of GCP service account. |
| `SMTP_HOST` | Optional | Default: `smtp.gmail.com` |
| `SMTP_PORT` | Optional | Default: `587` |
| `SMTP_USER` | Optional | For email verification |
| `SMTP_PASS` | Optional | Gmail App Password |
| `FROM_EMAIL` | Optional | Sender address |
| `APP_URL` | Optional | Base URL for email links |

## Local Development

```bash
# Install dependencies
pip install -r requirements.txt

# Create .env
echo "SECRET_KEY=localdev123" >> .env
echo "DATABASE_URL=postgresql://postgres:postgres@localhost:5432/dsa_tracker" >> .env

# Run Flask
python app.py

# Run scheduler manually
python scheduler.py
```

This project now requires PostgreSQL everywhere, including local development —
there is no SQLite fallback. Install PostgreSQL locally (or use Docker) and set
`DATABASE_URL` before running the app or scheduler; `db.py` raises a clear error
if it isn't set.
